## ** 2. The Demo Application **

This section describes the requirements and challenges of the demo application we want to build.

### Excel based Energy Report

The Excel file contains energy consumption data in tabular format. The columns include Date, Consumption, Peak Time, and other relevant information. The
challenge is to get an overview what the data is all about and understand the semantic meaning of the individual columns, so that the appropriate ones are
displayed in the frontend application.

### Converting Excel to JSON

In comparison to Excel, JSON is by far more commonly used in Angular development and therefore we want to preprocess the Excel file and convert it to json
format upfront. Developers have to know about excel parsing frameworks, data formats and json libraries, all of this with a specific technology stack (bash,
node, java, etc.). It also helps to show the power of generative AI in automating such tasks.

### Angular 19 Application

The frontend application itself should read the data from the backend (in this case a local json file) and use an Angular Material table to display it.
Additionally, we want to visualize the energy consumption trends over time using a line chart. For that developers have to initialize a workspace, generate the
application, create components, add libraries (material, charting), build and run the app.
